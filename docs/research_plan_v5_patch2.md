# 研究計画書 v5 パッチ2 — 解釈可能性と長距離依存の明示化

## 目的

PolyG (2025/04) と CausalRAG (2025/03) との **表面的類似性** に対応するため、
v5 計画書の主張の重心を以下2点に向けて移動する:

1. **長文書内における長距離議論依存の接続**
2. **推論経路の構造的可視化（説明可能性）**

両者とも既に実装済み (`CrossChunkEdgeExtractor`、議論経路を返す `GraphTraverser`)
の機能だが、計画書上の「売り」として前面に出ていない。以下の追加・修正で
重心を移す。

---

## 変更1: §1 「背景」に説明可能性の問題提起を追加

### 追加文（§1 の末尾に追記）

さらに、類似度ベースRAGは **なぜその答えを出したか** の構造的説明を提供しない。
ユーザーは取得された文のリストから推論経路を自分で再構成する必要があり、
特に長文書（科学論文・財務文書）では **CLAIM と EVIDENCE が数ページ離れて散在する**
ため、検証コストが大きい。

この観察から、問題の本質を2つに整理する:

* (a) 類似度だけでは **役割的関係**（主張・根拠・前提・対比）を区別できない
* (b) 類似度だけでは **長距離の議論依存** を接続できず、結果として **推論経路を明示化できない**

## 変更2: §2.4（新設）長距離議論依存の接続

### 新規サブセクションを §2.3 の直後に挿入

**§2.4 長距離議論依存の接続**

長い論文では CLAIM が p.2、EVIDENCE が p.8 に散在することが多い。
従来 RAG はチャンク単位の類似度で独立にスコアリングするため、CLAIM を含む
chunk は取れても、遠く離れた EVIDENCE を取り逃すことが構造的に避けられない。

議論グラフは chunk 境界を越えてエッジを張ることで、この問題を直接解く:

```
[chunk 2: CLAIM「Q3売上は12%減」]
         ▲
         │ SUPPORTS  (cross-chunk edge)
         │
[chunk 7: EVIDENCE「電子部品の在庫調整が主要因」]
```

実装（`CrossChunkEdgeExtractor`）の流れ:

1. 全ノードを埋め込みベクトル化（E5-Mistral-7B）
2. 各ノードの上位 top-k 近傍（類似度 ≥ 0.45）を候補に
3. 同一 chunk 内は除外（`EdgeExtractor` で処理済みのため）、既存辺も除外
4. 候補ペアのみ LLM `EdgeExtractor` で型付きエッジとして判定

これにより、長文書内の「離れた根拠」を **O(N²) の LLM 判定を回避しつつ**
自動接続できる。

## 変更3: §3 既存研究テーブルに PolyG / CausalRAG を追加

### 現行テーブルに2行を追加

| 研究 | アプローチ | 提案との違い |
| :---- | :---- | :---- |
| **PolyG (2025/04)** | KGQA の質問を4分類し、LLM が動的に Cypher を生成 | **対象が知識グラフ**であり文書RAG ではない。議論役割タクソノミーを持たず、長距離依存接続の概念もない |
| **CausalRAG (2025/03)** | 文書から LLM で **因果** グラフを構築、因果経路を辿る | **因果関係のみ**。支持 / 前提 / 対比などの rhetorical role を区別せず、cross-chunk の議論依存接続は扱わない |

### 新規貢献3点（§3末尾）の書き換え

v5 の新規貢献①②③を以下で置換する:

**① 文書議論グラフ（7役割×6関係）を基盤とする RAG 検索機構。**
PolyG は KG 上、CausalRAG は因果グラフ上の検索。本研究は **文書の議論役割グラフ**
を substrate とし、両者と直交する。

**② チャンク境界を越えた長距離議論依存の接続機構。**
類似度ベース RAG が原理的に扱えない「長文書内で数ページ離れた CLAIM–EVIDENCE
関係」を、埋め込みによる候補選択 + LLM による型付きエッジ判定の2段で自動接続
する。PolyG も CausalRAG もこの問題設定自体を扱わない。

**③ 推論経路としての構造化された説明。**
取得結果は単なる文のリストではなく、 **入口ノード → 型付きエッジ列 → 到達ノード**
という経路として提示される。精度だけでなく **人間が検証可能な推論トレース**
を主要な出力として位置付ける。

※ 旧①（クエリ型適応探索）は廃止ではなく、②③を実現するための下位機構として
位置付け直し、主張の重心から外す（PolyG と直接殴り合わないため）。

## 変更4: §4.3 評価指標に「trace quality」を追加

### 現行テーブルに1行を追加

| 指標 | 何を測るか | 本手法が有利と予想される理由 |
| :---- | :---- | :---- |
| **trace quality** | 取得された経路が「妥当な議論構造」として読めるか（1–5） | 役割付きエッジで構成された経路は人手で検証可能。類似度ベースの flat リストにはこの構造がない |

#### 評価方法

LLM-judge で各質問の取得経路を 1–5 で採点。加えて、小規模サブセット
（N = 50–100）で **人手評価** を実施し、LLM-judge との相関を報告する。
これは §9 の「構造化された説明可能性」の貢献を直接測る指標となる。

## 変更5: §4.4 アブレーションに2項目を追加

### 現行テーブルに追記

| 優先 | 実験 | 目的 |
| :---- | :---- | :---- |
| ★★★ | **CrossChunk ON/OFF + 閾値感度（0.35, 0.45, 0.55, 0.65）** | 長距離依存接続（新規貢献②）の寄与を単独で測定。閾値 0.45 の妥当性を感度分析で検証し、magic number 批判を回避 |
| ★★ | **単一 chunk 質問 vs 複数 chunk を跨ぐ質問での性能差** | 質問を「単一 chunk 内で回答可能」と「複数 chunk を跨ぐ必要がある」に二分し、後者で本手法の優位が顕著であることを示す。PolyG との差別化（長距離依存）の実証 |

これらは §2.4 および新規貢献② を **直接立証する** ablation であり、
この2項目が無ければ長距離依存の主張は空理論になる。優先度は最高レベル。

## 変更7: §9 期待される貢献の書き換え

### 現行全文を以下で置換

本研究は文書議論グラフを基盤とし、以下3点を提案・実装・評価する:

## 本パッチのサマリ

* **追加する主張**: 長距離依存接続、構造化された説明可能性
* **弱める主張**: クエリ型適応探索の新規性（②③の下位機構に位置付け直し）
* **追加する ablation**: CrossChunk ON/OFF + 閾値感度、単一/複数 chunk 質問分割
* **追加する指標**: trace quality（LLM-judge + 人手評価）
* **関連研究追加**: PolyG (2025/04), CausalRAG (2025/03)

狙いは **PolyG と直接殴り合わず、別軸で価値を提示する** こと。既存実装で
既に動いている機能（`CrossChunkEdgeExtractor`、経路付き取得）をフル活用する
ため、追加実装コストは ablation 2種と trace quality 評価スクリプトのみ。
